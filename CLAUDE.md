## What This Project Is

**Specter-Playground** is the development environment for the Specter DIY hardware
Bitcoin wallet. It contains:
- **MockUI**: A MicroPython + LVGL simulator for the updated device UI; not yet connected to real business logic;, runnable on Linux/macOS
- **Firmware source**: `src/` — Old Bitcoin wallet logic incl. old GUI, keystore, hosts, apps; Functionality shall be ported over to MockUI in the future; better separation between GUI and business logic is expected in the new architecture
- **Bootloader**: `bootloader/` — secp256k1-verified secure bootloader for STM32F469
- **Build toolchain**: Makefile targets for simulator, firmware, and i18n pipeline

## MCP Servers

Two MCP servers are available:

### lvgl-sim (TCP :9876)
Controls the running LVGL/MockUI simulator. Tools:
- `start_simulator` / `stop_simulator`
- `get_widget_tree` — JSON dump of current LVGL widget tree
- `click` / `long_press` / `swipe` — interact with widgets
- `get_state` / `set_state` — read/set SpecterState fields
- `screenshot` — capture current screen to `/tmp/sim_screenshot.png`

Start it: `make simulate` (keep running in background)

## Essential Commands

Build targets require the Nix dev shell (`nix develop -c ...`) for the cross-compiler
and toolchain. Python/pytest targets run directly in the `.venv`.

```bash
# Build (need nix shell)
nix develop -c make simulate              # start simulator (leaves it running)
nix develop -c make unix                  # build micropython_unix
nix develop -c make mockui ADD_LANG=de    # full firmware build, incl. German language file
nix develop -c make build-i18n ADD_LANG=de  # compile translation .bin files
nix develop -c make sync-i18n            # scan source for t("KEY") calls; update lang JSON files; regenerate translation_keys.py

# Tests (plain .venv, no nix needed)
pytest                                    # run all tests
pytest -k <name>                          # run specific test
```

`ADD_LANG` accepts a comma-separated list (e.g. `ADD_LANG=de,fr`). English is always
built; list only the additional languages needed.

## Absolute Rules

1. **Security path gate**: before editing `src/keystore/`, `src/rng.py`, or any
   file under `bootloader/core/` or `bootloader/keys/`, load and apply
   `.bmad/agents/security.md` in full. Never self-approve a HIGH risk finding.

2. **Tests first**: write a failing pytest test before any implementation change.

3. **i18n always**: all user-visible strings use `t("KEY")` via `I18nManager`.
   Add new keys to `scenarios/MockUI/src/MockUI/i18n/languages/specter_ui_en.json`,
   then run `nix develop -c make sync-i18n` to regenerate `translation_keys.py`.
   **Never edit `translation_keys.py` directly — it is auto-generated.**

4. **Manifests**: `manifests/mockui-shared.py` freezes all of `scenarios/MockUI/src/`
   recursively — no per-file listing is needed for files within that tree. Only update
   a manifest when adding code *outside* an already-frozen directory.

5. **Auto-generated files**: whenever a build step, tool, or script produces output
   that must not be committed, add a **path-independent** pattern to `.gitignore`
   (e.g., `lang_*.bin` instead of `build/flash_image/i18n/lang_en.bin`).

6. **MicroPython constraints**: no list comprehensions in hot paths, use `const()`,
   use `.format()` not f-strings where heap matters. See
   `.bmad/specialists/micropython-specialist.md` for the full constraints table.
